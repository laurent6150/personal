## 핵심 내용 요약

### 문제 인식
- McKinsey 연구: 강력한 의사결정 프로세스를 가진 조직이 평균 이상 매출 달성 가능성 **2배**
- 2026년: AI가 의사결정의 "공동 결정자"로 부상
- 미들 매니저 역할 변화: 업무 감독자 → 전략적 실행자(strategic enabler)

### 핵심 메시지
1. **문제 유형에 맞는 프레임워크 선택**: Cynefin으로 상황 진단 후 접근법 결정
2. **AI는 OODA 루프의 증폭기**: Observe/Orient 단계를 가속화
3. **권한 명확화가 병목 해소의 핵심**: RAPID으로 "누가 최종 결정자인지" 정의

### 주요 개념/프레임워크

#### 1. Cynefin Framework (Dave Snowden)

| 영역 | 특징 | 접근법 |
|------|------|--------|
| **Simple (명확)** | 원인-결과 관계 명확 | Sense → Categorize → Respond |
| **Complicated (복잡)** | 분석 필요, 전문가 영역 | Sense → Analyze → Respond |
| **Complex (복합)** | 패턴이 사후에만 보임 | Probe → Sense → Respond |
| **Chaotic (혼돈)** | 즉각 행동 필요 | Act → Sense → Respond |

**핵심**: 문제 해결 전에 "이 문제가 어떤 영역인가?"를 먼저 진단

#### 2. OODA Loop (John Boyd)

```
Observe → Orient → Decide → Act → (반복)
  관찰      판단      결정     행동
```

**2026년 변화**: AI가 Observe/Orient 단계를 가속화
- 실시간 사기 탐지
- 공급망 조정
- AI가 공동 결정자 또는 주도권 가질 수 있음

#### 3. RAPID Framework (Bain & Company)

| 역할 | 설명 |
|------|------|
| **R**ecommend | 제안 담당 |
| **A**gree | 동의 필요 (거부권) |
| **P**erform | 실행 담당 |
| **I**nput | 의견 제공 |
| **D**ecide | **최종 결정권자** |

**적합한 상황**: 대규모 조직, 의사결정 병목이 자주 발생할 때

#### 4. Vroom-Yetton-Jago Model

혼자 결정할지, 팀과 함께 결정할지 판단:
- "내가 결정하고 나중에 알린다" ↔ "함께 해결책을 찾는다"
- 5가지 접근법 중 상황에 맞게 선택

## 상세 내용

### 2026년 리더십 트렌드

**AI와 의사결정**:
"AI는 리더에게 속도와 명확성을 준다. 특히 AI를 의사결정을 **대체**하는 것이 아닌 **강화**하는 도구로 이해할 때. 번성하는 리더는 AI를 위협이 아닌, 더 빠르고 더 많은 정보로 더 나은 결정을 돕는 도구로 본다."

**미들 매니저의 진화**:
- 과거: 업무 감독자 (task supervisor)
- 2026년: 전략적 실행자 (strategic enabler) - 조직 우선순위를 현장에서 실현하는 사람

**자율성과 분산 책임**:
더 많은 조직이 리더리스 팀, 공유 소유권, 분산 책임을 실험. 리더의 역할이 "사람을 지시하는 것"에서 "시스템을 가능하게 하는 것"으로 전환.

## 활용 사례

### 원문 사례
- **Cynefin 적용**: 신제품 출시(Complicated) vs 시장 위기 대응(Chaotic)에 다른 접근
- **RAPID 적용**: 대기업에서 부서 간 의사결정 권한 명확화

### 적용 아이디어
- **나인투원 운영 의사결정**: Cynefin으로 문제 유형 진단 → 적합한 프로세스 적용
- **긴급 상황 대응**: OODA 루프로 빠른 관찰-판단-결정-행동 사이클
- **크로스팀 프로젝트**: RAPID으로 역할 명확화, "누가 D인가?" 정의

## 실천 포인트

### 즉시 적용 (5분)
- 현재 직면한 문제 하나를 Cynefin 4영역 중 어디인지 분류해보기
- "이 결정의 D(최종 결정자)는 누구인가?" 자문하기

### 단기 적용 (이번 주)
- 팀 미팅에서 RAPID 역할 실험 (작은 결정부터)
- 긴급 이슈에 OODA 루프 적용해보기

### 장기 고려
- 팀의 의사결정 프로세스 표준화 (어떤 상황에 어떤 프레임워크)
- AI를 Observe/Orient 단계에 활용하는 방안 검토
- 의사결정 권한 매트릭스 작성
