## 핵심 내용 요약

### 문제 인식
- 2025년 Andrej Karpathy가 명명한 "Vibe Coding"이 올해의 단어로 선정
- 1년 후, Karpathy가 후속 개념 "Agentic Engineering" 제안
- 전 세계 코드의 **41%가 이미 AI로 생성**되는 시대

### 핵심 메시지
1. **핵심 구분: 코드를 쓰는 것 → 에이전트를 오케스트레이션**: "당신은 코드를 쓰지 않는다—코드를 쓰는 에이전트를 지휘한다"
2. **AI는 기존 역량의 증폭기, 대체재가 아님**: 제트 엔진이 항공 경제학을 바꾼 것처럼, 에이전틱 워크플로우가 코드 작성/테스트/배포의 경제학을 바꿈
3. **빠른 피드백 루프가 핵심**: 빠른 컴파일, 빠른 테스트, 빠른 도구 응답이 에이전트 워크플로우의 생산성 결정

### 주요 개념/프레임워크

#### Vibe Coding vs Agentic Coding

| 구분 | Vibe Coding | Agentic Coding |
|------|-------------|----------------|
| **철학** | 창의적 흐름 유지 | 위임과 감독 |
| **접근** | 프롬프트 → 실험 → 코드 정제 | 목표/제약/품질기준 정의 → 에이전트 작업 |
| **개발자 역할** | 대화형 협업자 | 아키텍트/오케스트레이터 |
| **적합 상황** | 컨셉 테스트, 유연성 필요 시 | 구조화된 엔터프라이즈 워크플로우 |
| **LLM 취급** | 고급 자동완성 | 목표 지향적 에이전트 팀 |

#### 2026 추천 도구 스택
| 용도 | 추천 도구 |
|------|----------|
| 입문/범용 | GitHub Copilot (광범위한 IDE 통합) |
| Vibe Coding | Cursor (실시간 협업 최적화) |
| 복잡한 자율 작업 | Claude Code (에이전틱 워크플로우) |

## 상세 내용

### Agentic Engineering의 핵심 원칙
LLM을 "고급 자동완성"으로 취급하는 대신:
1. **목표(Goals)** 정의
2. **제약조건(Constraints)** 설정
3. **품질 기준(Quality Criteria)** 명시
4. **워크플로우** 설계

→ 각자 전문 역량을 가진 AI 에이전트 팀이 인간의 방향 아래 코드 생성, 테스트, 디버깅, 통합

### 2026 베스트 프랙티스
1. **스니펫이 아닌 시스템 구축**: 모듈이 장기 아키텍처에 어떻게 맞는지 고려
2. **가드레일 설정**: AI 에이전트가 수정할 수 있는 범위 제한
3. **감독 유지**: 배포 전 생성된 코드 항상 리뷰
4. **컨텍스트 제공**: 자연어 프롬프트에 상세한 입력 제공
5. **아키텍트로 행동**: 라인 편집 대신 명확성, 구조, 전체 설계에 집중

### 하이브리드 접근법이 정답
- **Vibe Coding 사용**: 컨셉 테스트, 유연성 필요 시
- **Agentic Systems 사용**: 구조화된 엔터프라이즈급 워크플로우
- **두 접근법 혼합**: 속도와 통제 모두 필요한 팀
